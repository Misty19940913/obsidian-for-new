## 最常用的 Markdown（15個）

### 每天都用（5個）
```
**粗體**     *斜體*     # 標題     - [ ] 待辦     [[連結]]
```

### 需要時查詢（10個）
```
---  （水平線）    - 項目    （無序列表）
1. 第一  （有序）    ![[圖片.png|300]]  （圖片）
> [!tip]  （callout）    `代碼`     | a | b |   （表格）
![[筆記名]]  （嵌入）    ~~刪除~~    （刪除線）
[文字](URL)   （外部連結）
```

## 資料遷移：常見工具移入 Obsidian

| 來源 | 方法 | 連結語法 |
|------|------|---------|
| **Notion** | 匯出 Markdown → 複製到 Vault → 手動加 `[[]]` | 需重建 |
| **Evernote** | `.enex` 匯出 → `evernote-to-md` 轉換 | 需重建 |
| **Roam** | Export Markdown → 直接複製（語法相同）| 自動保留 |
| **Logseq** | Export Markdown → 直接複製（語法相同）| 自動保留 |

> [!tip]+ 遷移後最重要的事
> 一次不要遷移所有筆記。用「80/20 法則」：先遷移被多次引用的 20%，其餘慢慢來。

## 學習进度：你在哪個階段？

| 等級 | 階段 | 你能做到的事 |
|------|------|-------------|
| **L1** | 新手剛接觸 | 會建立筆記、會用基本 Markdown |
| **L2** | 基礎操作 | 會建立連結、會用搜尋、會安裝插件 |
| **L3** | 中級應用 | 有個人系統（PARA/MOC）、會用模板 |
| **L4** | 進階整合 | Git 備份、Dataview 查詢、接入了 AI |
| **L5** | 專家級 | 有完整第二大腦、持續產出 |

## 克服三個常見障礙

| 障礙 | 解法 |
|------|------|
| **不斷設定工具、忘了寫筆記** | 每新增一個插件，先問「這個解決什麼問題」|
| **無法持續寫作** | 從「最小承諾」開始：每天只寫 3 行 |
| **舊筆記太多無從整理** | 先處理最常被引用的 20%，其餘當歷史資料庫|
