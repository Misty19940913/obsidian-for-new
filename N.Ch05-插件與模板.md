# N.Ch05｜插件與模板：第一二週配置

> [!info]+ 這章結束時你會做到的事
> 1. 安裝第一二週用得上的 4 個核心插件
> 2. 使用模板提升筆記建立效率
> 3. 知道哪些插件組合會讓 Obsidian 變慢（必須避免）

---

## 如何安裝社群插件

1. 開啟 `設定` → `社群插件` → 啟用社群插件
2. 點 `瀏覽` → 搜尋名稱 → 點 `安裝`
3. 在 `已安裝` 區啟用

> [!warning]+ 安裝前先問自己四個問題
> 1. 插件有公開原始碼嗎？
> 2. 開發者是否活躍（最近三個月有更新）？
> 3. 安裝量多少？（越高越安全）
> 4. **我真的需要這個功能嗎？**
>
> 不需要的功能 = 潛在風險。備份從來不是選擇，是必須。

## 第一週：建立基礎配置（4個插件）

### 1. Templater —— 模板系統

讓每次新建筆記自動帶入預設格式。

**能做：**
- 動態插入日期、檔案名
- 自動建立資料夾結構
- 執行 JavaScript 片段

**基本設定：**
1. 安裝並啟用 Templater
2. `設定` → `Templater` → 設定模板資料夾位置（建議：`templates/`）
3. 在設定的資料夾裡建立模板檔案

**第一個模板：日誌模板（`daily.md`）**
*(註：請確認安裝的是社群插件 Templater，並且可以把原生的「核心插件 -> 模板」關閉以免混淆)*

```markdown
---
title: {{date}}
tags:
- 日誌
created: {{date}}
---

# {{date}}

## 今天發生什麼

## 學到了什麼

## 三個關鍵連結

- [[]]
- [[]]
- [[]]
```

---

### 2. Dataview —— 查詢引擎

用類似 SQL 的語法查詢你的筆記。

**你能做到的：**
- 「這週新建的所有筆記」
- 「所有含 `#目標` 標籤的筆記」
- 「我的專案進度」

**基本語法：**
\`\`\`dataview
TABLE file.name, file.ctime
FROM ""
WHERE contains(tags, "日誌")
SORT file.ctime DESC
LIMIT 10
\`\`\`

---

### 3. Tasks —— 任務管理

比原生 `- [ ]` 更強的任務時間管理。

你能做到的：
- `- [ ] 開會 ⏳ 2026-05-25`（截止日）
- `- [ ] 每週運動 🔁 every Monday`（重複任務）
- `- [ ] 已完成 ✅ 2026-05-20`（完成記錄）

> [!tip]+ 適合與 Dataview 一起用
> Tasks 管理任務語法，Dataview 負責查詢和視覺化。

---

### 4. Various Complements —— 輸入增強（可選，中文用戶推薦）

讓 `[[` 和 `#` 等符號自動補全，輸入更順暢。

## 第二週：提升效率的插件（3個，可選）

| 插件 | 用途 | 何時裝 |
|------|------|--------|
| **Kanban** | 在 Obsidian 裡做看板 | 需要專案進度視覺化時 |
| **Style Settings** | 調整介面字體、側邊欄寬度 | 對美觀有要求時 |
| **Iconize** | 為資料夾和筆記加圖示 | 想一眼區分不同類型時 |

---

## ⚠️ 插件衝突警告：這 3 個組合會讓 Obsidian 變慢

| 組合 | 問題 |
|------|------|
| Dataview + Templater + Tasks + Kanban 同時啟用 | 啟動慢、搜尋卡 |
| 多個插件同時讀寫同一個 vault 資料夾 | 同步衝突 |
| 安裝 15 個以上插件 | 記憶體佔用高 |

**建議：核心插件不超過 6-7 個，用不到的果斷停用。**

## 實戰模板：PARA 專案管理（帶有 MOC）

```markdown
---

phase: 策劃

status: active
tags: [專案, #產品]
---

# 產品Launch MOC

## 專案概覽

| 欄位 | 內容 |
|------|------|
| 階段 | {{phase}} |
| 截止日期 | {{deadline}} |
| 進度 | 20% |

## 階段追蹤

### 階段 1：策劃
- [x] 市場研究
- [ ] 用戶訪談

### 階段 2：執行
- [ ] 開發 MVP
- [ ] 測試驗證

## 相關筆記

[[策劃]]
[[推廣計畫]]
```
