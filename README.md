## 🎯 適合誰與專屬路徑

不同背景的人，使用 Obsidian 的重點完全不同：

### 1. 純新手（從未接觸過筆記軟體）
- **優先目標**：建立基本操作習慣，不要急著最佳化。
- **建議路徑**：前兩週每天只寫 1 篇筆記並練習建立 `[[連結]]`，不要碰任何插件。

### 2. 從 Notion / Evernote 跳槽者
- **優先目標**：改變分類習慣，重新建立「連結」思維。
- **建議路徑**：先遷移最核心的 20% 筆記，試著用 MOC 取代傳統的多層級資料夾。

### 3. 工程師 / 技術背景
- **優先目標**：把 Obsidian 整合到現有工作流。
- **建議路徑**：先用雲端設定好備份，然後直接探索 Templater 與 Dataview 打造自動化。

### 4. 創作者 / 研究者
- **優先目標**：快速捕捉靈感，系統化產出內容。
- **建議路徑**：設定每日捕獲（Daily Notes）流程，並用 Canvas 畫布策劃內容大綱。

## 🗺️ 學習路線圖：三個階段

```text
第一階段（第 1-4 週）：熟悉基本操作
  → 會下載、會建立 Vault、會寫 Markdown、會建立連結

第二階段（第 5-8 週）：建立個人系統
  → 選擇一個組織方法論（MOC 或 PARA）
  → 安裝並用起來 Templater / Dataview / Tasks
  → 設定每日 capture 流程

第三階段（第 9+ 週）：深化與系統化
  → 接入 AI 大模型
  → 整理並連結舊筆記，用 MOC 串聯
```

## 💡 核心心法

> **先寫再整理** — 不要先花兩小時設計資料夾分類。寫多了自然知道怎麼整理。

> **不要追求完美的筆記** — 一行字也值得記。筆記是給自己用的，不是作品。

> **養成連結的習慣** — 想到「這跟之前什麼有關」就順手 `[[]]`。
