# N.Ch02｜寫出第一篇筆記

> [!info]+ 這章結束時你會做到的事
> 1. 能夠用兩種方式建立新筆記
> 2. 掌握 15 個最常用的 Markdown 語法
> 3. 使用 Obsidian 專屬語法（連結、標籤、callout）
> 4. 下載並使用實作模板

---

## 建立筆記的兩種方式

### 方式一：直接新建

- 按 `Ctrl/Cmd + N`
- 或在檔案總管空白處右鍵 →「新建筆記」

### 方式二：透過連結自動建立

輸入 `[[筆記名稱]]`，如果筆記不存在，點一下就自動建立並連結。

**現在試著做：** 在任何一篇筆記裡，試著輸入 `[[我的第一篇測試筆記]]`，點下去看看。

## 📝 每天都會用到的語法（5個）

> 這 5 個一週後會變成肌肉記憶，不用刻意記。

| 語法 | 範例 |
|------|------|
| `**粗體**` | `**粗體**` |
| `*斜體*` | `*斜體*` |
| `# 標題` | `# H1`、`## H2`、`### H3` |
| `- [ ]` 任務 | `- [ ] 未完成`、`- [x] 已完成` |
| `[[連結]]` | `[[筆記名稱]]` |

| 刪除線 | `~~文字~~` |

## 🗂️ YAML Frontmatter：讓筆記有元資料

放在筆記最頂端，用三個 `---` 包住：

```yaml
---
title: 這是標題
tags:
- 標籤1
- 標籤2
created: 2026-05-28
---
```

只需要記這三個：`title`、`tags`、`created`。其他慢慢加。

---

## 💡 連結 vs 標籤：何時用哪個？

新手最常犯的錯就是把 Obsidian 當 Evernote 用，瘋狂打標籤。記住這個實用原則：

| 情境 | 該用什麼 | 範例 |
|------|---------|------|
| 這兩篇筆記有「邏輯關聯」 | **用 `[[連結]]`** (強關係) | `今天讀了[[原子習慣]]...` |
| 這篇筆記屬於「某個類別」或「狀態」 | **用 `#標籤`** (弱關係) | `#日記`、`#待辦`、`#靈感` |

> 優先使用 `[[連結]]` 來串聯知識，`#標籤` 只用來做大範圍的狀態過濾（建議總標籤數保持在 10 個以內）。

## ✋ 跟著做：建立你的第一篇日誌筆記

**步驟：**

1. `Ctrl/Cmd + N`，命名：`2026-05-28`
2. 複製貼上下列模板內容
3. `Ctrl/Cmd + S` 儲存

```markdown
---
title: 2026-05-28
tags:
- 日誌
created: 2026-05-28
---

# {{title}}

## 今天發生什麼

<!-- 在這裡寫 -->

## 學到了什麼

<!-- 在這裡寫 -->

## 三個關鍵連結

- [[]]
- [[]]
- [[]]
```

**驗證：做到這三步就算完成**

- [ ] 新建了一篇日期命名的筆記
- [ ] YAML 填好了 title / tags / created 三個欄位
- [ ] 在「今天發生什麼」下面寫了一行字
